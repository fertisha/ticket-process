<h1>ServiceNow Incident Ticket: Creation, Troubleshooting, and Resolution</h1>
This tutorial demonstrates how a Help Desk technician creates, documents, troubleshoots, and resolves a ServiceNow incident after receiving a support call from an end user.<br />


<h2>Environments and Technologies Used</h2>

- ServiceNow
- Windows 11
- Google Chrome
- Microsoft Outlook (Example user issue)


<h2>Operating Systems Used </h2>

- Windows 11</b> (21H2)

<h2>List of Prerequisites</h2>

Before you begin, make sure you have:

- A ServiceNow account
- A Windows 11 computer
- Google Chrome
- A user who needs IT support

<p>
  <h2>Step 1: Create ServiceNow incident</h2>
First, sign in to your ServiceNow account using your work email and password (Single Sign-On). After you sign in, you will see the main page where you can view your assigned work and your team's tickets.

This page helps you keep track of:

- The incidents assigned to you.
- The response time for your tickets (SLA).
- Tickets waiting to be assigned.
- Tasks assigned to you.

After receiving a call from a user who needs IT support, click the "+" button next to the Home tab to open a new tab and begin creating a new incident.

</p>
<br />

<p>

<img width="2560" height="1204" alt="image" src="https://github.com/user-attachments/assets/d4b6bd14-e330-4c02-ab45-1e5dac9453b7" />

<p>
  <h2>Step 2: Fill all the required fields</h2>
After opening a new incident, fill in the required information about the user's issue. Select the caller's name, enter a short description of the problem, and provide a more detailed explanation if needed.

Next, choose the appropriate State, Impact, Urgency, and Priority based on the issue. In this example, all values are set to Low because the issue is expected to be a quick and simple fix.

Once all the required information has been completed, click Assign to me. This will assign the incident to you, allowing you to begin troubleshooting and working toward a resolution.

</p>

<h2>Step 3: Troubleshoot the Issue</h2>

<p>
After assigning the incident, begin troubleshooting the user's issue.
</p>

<p>
In this example, the user reports that the classic Outlook ribbon is missing.
</p>

<h4>Option 1 - Guide the User</h4>

<p>
Explain over the phone how to restore the classic Outlook ribbon and confirm that the issue has been resolved.
</p>

<h4>Option 2 - Remote Support</h4>

<p>
If the issue cannot be resolved over the phone, ask the user for permission to start a remote support session using Microsoft Teams or BeyondTrust.
</p>

<ul>
<li>Identify the cause of the issue.</li>
<li>Restore the classic Outlook ribbon.</li>
<li>Confirm that the issue has been resolved.</li>
<li>Inform the user that the incident will be closed.</li>
</ul>
